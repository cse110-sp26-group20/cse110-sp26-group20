# Team Kickoff

- [ ] Conduct Group Bonding Activity: Do something relaxed and fun, preferably in-person, for at least an hour.
- [ ] Hold a Kickoff Meeting:
    - [ ] Discuss Team Charter topics (see Section 5 below).
    - [ ] Define a meeting structure (e.g., agenda review, key topics, takeaways).
    - [ ] Take Markdown meeting notes capturing: meeting type, attendance, time/place, agenda, future action items, and decisions (include reasoning).
    - [ ] Name the file mmddyy-topic.md (e.g., 040926-kickoff.md).
    - [ ] Save the file in the /admin/meetings/ directory.

# Team Branding & Page
- [ ] Develop Team Brand: Choose a team name and optional branding materials (logo, colors, etc.). Save these in /admin/branding.
- [ ] Set Slack Icon: Change your workspace icon to an image that represents your brand and prominently features your team number.
- [ ] Create Team Page (team.md):
    - [ ] Create the file at /admin/team.md.
    - [ ] Add a link to this file in your root README.md.
    - [ ] Include: team name, brand, core values, and a member roster (brief overview for each member + link to their personal GitHub page).

#Team Intro Video
- [ ] Record Video: Create a max 2.5-minute introductory video (MP4 or Quicktime).
- [ ] Upload Video: Name the file exactly teamintro.mp4 and save it at /admin/videos/teamintro.mp4.

# Team Charter (Discussion Points)
Discuss these during your Kickoff Meeting to align the team:

- [ ] Personal Goals: Outline individual expectations and how they benefit the team.
- [ ] Values & Mission: Define core principles, purpose, and shared milestones.
- [ ] Strengths & Weaknesses: Identify each member's skills and how to address team weak points.
- [ ] Roles & Accountabilities: Assign clear responsibilities (with some flexibility) and determine who handles logistics like meeting notes and task creation.
- [ ] Needs & Expectations: Share availability/schedules, define communication strategies, and establish a conflict resolution process.
- [ ] Rules & Activities: Agree on working methods, success metrics, and how to ensure fair contributions.
